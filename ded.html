DrugExposureDiagnostics

DrugExposureDiagnostics is a package to conduct Ingredient specific diagnostics for drug exposure records in the Observational Medical Outcomes Partnership (OMOP) common data model.

https://darwin-eu.github.io/DrugExposureDiagnostics/

Main functions you would need to interact with

Before running any drug utilisation study, it is paramount to examine the quality of your drug exposure data in your OMOP CDM. To support this, the OHDSI community has developed an open-source R package, DrugExposureDiagnostics.

There are two functions you often need to interact with in practice:

  1. To run the user-specified checks on drug information executeChecks().

  2. To viewing the results via interactive shiny viewResults().

Create a CDM reference

For this demonstration we will use the GiBleed dataset:

library(DrugExposureDiagnostics)
library(omock)
library(dplyr)
library(here)

cdm <- mockCdmFromDataset(datasetName = "GiBleed", source = "duckdb")
cdm$drug_exposure <- cdm$drug_exposure |>
  mutate(quantity = sql("floor(random() * 6) + 1")) |>
  mutate(quantity = case_when(
    quantity == 1 ~ 0,
    quantity == 2 ~ 1,
    quantity == 3 ~ 5,
    quantity == 4 ~ 10,
    quantity == 5 ~ 30,
    quantity == 6 ~ 100
  )) |>
  compute(name = "drug_exposure")
cdm

executeChecks()

executeChecks() runs a set of ingredient-level diagnostics on the OMOP CDM drug exposure data and returns a named list of diagnostic results.

?executeChecks()

Imagine you have a study centred around common pain-relief medications.

Suppose you want to investigate a simple and familiar ingredients: acetaminophen.

acetaminophenId <- 1125315L

Then you can look into checks including:

  1. their missingness (e.g., missing start date, missing quantity)
  2. their exposure duration
  3. the quantity of the drugs
  4. route (e.g., oral, tablet, topical)

Example code for executeChecks()

ingredient_checks <- executeChecks(
  cdm = cdm,
  ingredients = acetaminophenId,
  checks = c("missing", "exposureDuration", "route", "quantity"),
  minCellCount = 5, 
  earliestStartDate = NULL
)

What other checks can you do?

You may have noticed that from ?executeChecks() that these checks can be performed:

  • “missing”
  • “exposureDuration”
  • “type”
  • “route”
  • “sourceConcept”
  • “daysSupply”
  • “verbatimEndDate”
  • “dose”
  • “sig”
  • “quantity”
  • “daysBetween”
  • “diagnosticsSummary”

Visualise the results

You can produce a shiny to visualise the results using viewResults().

Visualise the results

First you would need to export the results.

writeResultToDisk(
  resultList = c(ingredient_checks),
  databaseId = "Eunomia",
  outputFolder = here(),
  filename = "DrugExposureDiagnosticsResults"
)

Now you should see a .zip file in your working directory.

Visualise the results

viewResults(dataFolder = here())

This will prompt you to install a couple packages. At the end you have a new directory where you can launch the shiny!
